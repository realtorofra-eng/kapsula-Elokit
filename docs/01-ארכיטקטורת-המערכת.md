# ארכיטקטורת המערכת — שפת הנשמה
### איך הכול בנוי, ולמה הכול קורה בתוך המערכת עצמה

עיקרון על: **מערכת אחת סגורה.** גוררים קבצים → הסוכנים רצים בפנים → שיעור יוצא.
אין יציאה לכלי חיצוני, אין העתק־הדבק, אין שרת לתחזק. קובץ אחד שנפתח בדפדפן.

---

## 1. מבט־על: הכול בקובץ אחד

```
                    ┌──────────────────────────────────────────────┐
                    │           שפת-הנשמה.html (המערכת)             │
                    │                                              │
   קבצי גלם  ───────▶│  גוררים קבצים   →   4 סוכנים רצים בפנים      │──▶  שיעור מוכן
   (PDF/וורד/        │  (בדפדפן)            חולץ→אדריכל→קולן→עורך    │     (נשמר בספרייה
    תמונה/טקסט)      │                          ▲                  │      שבמחשב)
                    │                   פרופיל הקול (בתוך המערכת)   │
                    └──────────────┬───────────────────────────────┘
                                   │ קריאה ישירה (HTTPS)
                                   ▼
                          מנוע הבינה (Claude API)
                       הדבר היחיד מבחוץ — מפתח פעם אחת
```

## 2. החלקים

### א. אחסון — בתוך הדפדפן (localStorage)
כל מה שהמערכת יוצרת ויודעת נשמר מקומית במחשב שלך, לא בענן:
- **ספריית השיעורים** — כל שיעור מוגמר, לפתיחה/הורדה בכל רגע.
- **פרופיל הקול** — הטקסט שמלמד את המערכת איך את מדברת.
- **המפתח וההגדרות** — נשמרים אצלך בלבד.

> רוצה גם סדר בקבצי הגלם עצמם? יש תיקיית `שיעורים/` עם תיקייה לכל שיעור. אופציונלי.

### ב. עיבוד — ארבעת הסוכנים, רצים בתוך המערכת
כשלוחצים "בנה את השיעור", המערכת מריצה ארבע קריאות בזו אחר זו, כל אחת בונה על הקודמת,
וכולן מקבלות את **פרופיל הקול** כהוראת־על:
1. **החולֵץ** — קורא את כל הקבצים (PDF ותמונות נקראים ישירות ע"י המנוע; וורד/טקסט מחולצים בדפדפן), ומוציא מושגים, מבנים, שיטות.
2. **האדריכל** — מזהה את נשמת השיעור ובונה קשת לב→שכל.
3. **הקולָן** — כותב את השיעור המלא בקול שלך.
4. **העורך** — בודק מול ה‑Checklist (לב/קול/קרקע/חיבור) ומפיק תוכנית סופית.

את רואה כל שלב מתקדם על המסך, ויכולה לפתוח את הפלט של כל סוכן.

### ג. הצגה — מתוך התוצר
תוכנית השיעור כוללת כבר **שקופיות מוצעות** ו**הערות דובר**. ממירים למצגת עם
מסך נקי לתלמידות + מסך פרטי לך. פירוט: [תשתית-מצגות](../המערכת/תשתית-מצגות.md).

## 3. הדבר היחיד מבחוץ: מנוע הבינה
אי אפשר להפעיל בינה מלאכותית בלי מנוע. המערכת קוראת ישירות ל‑Claude API
(דרך הדפדפן, בלי שרת ביניים). מזינים מפתח **פעם אחת** ב"הגדרות", הוא נשמר אצלך,
ומשם — אף פעם לא יוצאים מהמערכת.

**עלות:** משלמים לפי שימוש (טוקנים) ישירות ל‑Anthropic. שיעור בודד עולה סנטים בודדים.
ברירת המחדל היא המנוע האיכותי (Opus) לשמירה על הקול; אפשר לעבור ל‑Sonnet החסכוני בהגדרות.

---

## 4. למה זה מתאים למורָה לא־טכנית
- **קובץ אחד.** דאבל־קליק, נפתח בדפדפן. אין התקנה, אין קוד, אין שרת.
- **הכול במקום אחד.** קבצים, סוכנים, תיקון, ספרייה — מסך אחד.
- **הקול נשמר ומשתפר.** עורכים את פרופיל הקול פעם אחת, וכל שיעור עתידי נשמע כך.
- **שום דבר לא הולך לאיבוד.** הכול נשמר אצלך במחשב; יש כפתור גיבוי.

## 5. חבילת השיעור — חמישה תוצרים בתיקייה אחת
לכל שיעור, כפתור **📦 חבילת שיעור מלאה** בונה ומוריד תיקייה אחת (ZIP) בשם `שיעור NN - שם`:
```
שיעור 01 - מבוא למבטולוגיה/
  מערך-שיעור.md / .html   ← השיעור המלא: טקסט, דוגמאות, תרגילים משולבים
  מצגת.html               ← מצגת מעוצבת + שקופית קישור לתרגילים (S להערות)
  תרגילים.md / .html      ← קובץ תרגילים נפרד (להדפסה/PDF), בקול של עופרה
  שיעורי-בית.html         ← להדפסה כ‑PDF לתלמידות
  סיכום-וואטסאפ.txt       ← הודעה קצרה לשליחה לקבוצה
  עזרים/                  ← תמונות שנוצרו + רעיונות-וידאו.txt
  קרא-אותי.txt
```
כל קובץ נוצר **בקול של עופרה** דרך אותם סוכנים ופרופיל הקול. התרגילים נמשכים מהחומר ומותאמים לשפה שלה (דוגמאות מהחיים, משלים, סיפורי מקרה), ומשולבים בשיעור ובקובץ נפרד עם קישור מהמצגת.

## 6. הצגה, לוגו וחיבור
- **מצב הצגה** מובנה: מסך קהל נקי + הערות דובר פרטיות **הניתנות לעריכה חיה**, טיימר, קישור לתרגילים, וייצוא.
- **הלוגו** של עופרה (אותיות בעיגול, רקע שקוף) משובץ בכל המערכת — כותרת, מצגות, צ'אט, favicon.
- **בדיקת חיבור** בהגדרות מוודאת שמפתח ה‑Claude מחובר ועונה.
- **עוזרת ההוראה** (צ'אט) פועלת בארבעת המצבים: שותף יצירתי · מבצע מדויק · מתרגם שפה · מנהל פרויקט.

## 7. איך זה גדל
- **1–20 שיעורים:** עובד מצוין כמו שהוא. הספרייה מתמלאת.
- **המשך:** ככל שתעדכני את פרופיל הקול עם ניסוחים מוצלחים שלך, המערכת תישמע מדויקת יותר ויותר — היא לומדת את הקול שלך דרכך.

> מבנה הקבצים בפרויקט: `index.html` (האפליקציה) · `api/claude.js` (פרוקסי לשרת) · `logo-data.js` + `assets/` (לוגו) · `docs/` (מסמכי הקול, המצגות, ההנחיות) · `agents/` (הסבר הסוכנים) · `מדריך.html` (מדריך שלב-אחר-שלב) · `פריסה-ל-Vercel.md`.
